## 2025-05-15 - Redundant API calls in get_meter_details **Learning:** The internal `get_meter_details` function automatically called `get_meter_gsp`, which makes an additional API request. This happened even when calling `get_consumption`, which doesn't need GSP data, and even within `get_meter_gsp`'s own default argument, causing circular (though limited) redundant calls. **Action:** Added an `include_gsp` parameter to `get_meter_details` (defaulting to `TRUE` for backward compatibility) and set it to `FALSE` in `get_consumption` and `get_meter_gsp` to eliminate unnecessary network overhead.

## 2025-05-15 - Significant overhead of ifelse() on consumption data **Learning:** Using `ifelse()` to replace NAs in consumption data frames was significantly slower and more memory-intensive than logical indexing. In benchmarks with 1M rows, `ifelse` took ~87ms and 53MB, while logical indexing took ~13ms and 15MB. **Action:** Replaced `ifelse(is.na(x), 0, x)` with `x[is.na(x)] <- 0` in `combine_consumption`.
