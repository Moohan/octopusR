## 2025-05-15 - Redundant API calls for GSP
**Learning:** The package was performing redundant API calls to retrieve the Grid Supply Point (GSP) because `get_meter_details` always called `get_meter_gsp` when meter details were missing, and `get_meter_gsp` also had a default parameter that called `get_meter_details` recursively. Furthermore, `get_consumption` called `get_meter_details` even when GSP wasn't needed for its operation.
**Action:** Added an `include_gsp` parameter (default `TRUE`) to `get_meter_details` and switched from `ifelse` to an explicit `if/else` block to allow skipping the GSP retrieval. Updated `get_meter_gsp`'s default parameter and `get_consumption`'s internal call to use `include_gsp = FALSE`, eliminating the redundancy and resulting in a ~16x speedup for `get_meter_details` (when skipping GSP) and saving one network request per `get_consumption` call.
